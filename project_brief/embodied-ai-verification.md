# 具身智能验证协议 — Embodied AI Verification Protocol

> 文件版本: N32 · 最后更新: 2026-07-06
> 验证对象: MotionPlanner (10 种硬编码模板) + MotionTranslator (LLM 任意动作翻译)

---

## 一、背景：为什么需要验证？

### 技术现状

```
MotionPlanner.PlanFromDescription():
  ├─ 硬编码模板 (10种): 挥手 / 点头 / 摇头 / 鞠躬 / 伸懒腰 / 叉腰 / 捂脸 / 指 / 招手 / 合十
  └─ 回退 → 泛用微动: 头左右晃 3° + 眼珠微动 (仅 2 帧，极简)

MotionTranslator.TranslateAsync():
  拦截泛用微动回退 → 调用 DeepSeek API → 生成结构化关键帧
```

### 核心验证问题

> "AI 是真的理解了动作描述并生成了合理的参数序列，
> 还是只是随机生成了一些看起来"有变化"的参数组合？"

### 验证标准

| 等级 | 定义 | 判定方法 |
|------|------|---------|
| **PASS** ✨ | 动作可被肉眼识别且符合语义 | 人类观察 + GLM-4V 对比 |
| **PASS** ✅ | 参数序列在物理上合理（范围/对称/平滑） | 自动校验 |
| **FAIL** ❌ | 参数越界/无意义抖动/不符合语义 | 自动校验 |

## 三、自动化验证工具

### 3.1 MotionVerifier.cs

验证核心文件。位置: `Assets/Scripts/Live2DFramework/ActionAgent/MotionVerifier.cs`

**功能**:
1. 逐条执行测试集中的描述
2. 记录: 是否走 LLM 路径、关键帧数、持续时长、使用的参数名集合
3. 物理合理性校验:
   - 参数值是否在映射范围内
   - 左右对称参数是否成对出现（眼/眉）
   - 是否产生"无意义抖动"（相邻帧微小差异）
   - 最后一个关键帧是否归零
4. 输出结构化验证报告

**接口**:
- `RunVerificationSuite()` → 返回完整验证报告 (string)
- `RunSingleTest(string description)` → 返回单条结果
- `CompareMotionPlans(MotionPlan a, MotionPlan b)` → 对比两个动作的相似度

### 3.2 验证工具注册

在 ToolCallInvoker 注册 `run_verification` 工具：
- 调用 `MotionVerifier.RunVerificationSuite()`
- 返回结构化报告给 DeepSeek
- 可指定 `--tier=1`, `--tier=2` 运行不同层级的测试

### 3.3 GLM-4V 交叉验证

对于每条测试，自动走 self_review 路径：
1. 触发动作
2. 截图
3. 发 GLM-4V: "请仅用文字描述这张图片中人物的姿态，不要评价" → 得到纯描述
4. 判断 AI 的描述是否匹配原始动作描述

## 四、验证流程

### 4.1 每日验证（自动化，~2 分钟）

```bash
# 在 Unity 编辑器中运行
Tools → AI → 运行具身智能验证
```

输出报告到 `project_brief/verification_report_YYYY-MM-DD.md`

### 4.2 每周验证（人工辅助，~15 分钟）

1. 运行完整测试集（15 条）
2. 目测 T1~T10 每个动作截图
3. 填写肉眼识别度评分: [1~5]
4. 记录到验证报告

### 4.3 回归测试

每次修改 MotionTranslator.cs / MotionPlanner.cs / ToolCallInvoker.cs 后运行一次

## 五、验证指标

### 5.1 自动指标

| 指标 | 计算方法 | 目标值 |
|------|---------|-------|
| **LLM 触发率** | LLM 路径次数 / 总测试数(非硬编码) | 100% |
| **平均关键帧数** | 所有 LLM 路径的关键帧数均值 | ≥ 5 |
| **参数合规率** | 值在范围内的参数比例 | 100% |
| **对称配比率** | 成对参数同时出现的比例 | ≥ 70% |
| **归零率** | 含归零帧的动作比例 | ≥ 80% |

### 5.2 人类评分指标

| 指标 | 定义 | 评分 |
|------|------|------|
| **识别度** | 肉眼能否猜出意图动作 | 1~5 |
| **自然度** | 动作是否流畅自然 | 1~5 |
| **表现力** | 是否有多部位协同（不只有头动） | 1~5 |

### 5.3 通过标准

| 等级 | 自动指标 | 人类评分 |
|------|---------|---------|
| 🥇 **完全通过** | 全部达标 | 平均 ≥ 4 |
| 🥈 **基本通过** | 自动指标 ≥ 80% | 平均 ≥ 3 |
| 🥉 **需改进** | 自动指标 ≥ 60% | — |
| ❌ **未通过** | 自动指标 < 60% 或有崩溃 | — |

## 七、验证脚本用法

### 在 Unity 中验证

1. 打开 Unity 编辑器，启动场景
2. 打开控制台 (Ctrl+Shift+C)
3. 在聊天输入框输入:
   - `"害羞地捂脸"` → 观察自动播互动
   - `\verify` → 运行完整验证套件
4. 或通过代码:

```csharp
// 在任意协程中：
var verifier = new MotionVerifier();
string report = verifier.RunVerificationSuite(renderer.Mapper, renderer.CubismModel);
Debug.Log(report);
```

### 在右键菜单中添加验证按钮（可选）

ContextMenu 新增「🧪 具身验证」按钮 → 一键运行 T1~T10
